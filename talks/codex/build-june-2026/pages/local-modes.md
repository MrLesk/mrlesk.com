# Local work modes

<div class="cards-grid cols-4" style="margin-top: calc(20 * var(--pt));">

<div class="card">
  <div class="card-eyebrow">Local threads</div>
  <div class="card-body">Keep the conversation, commands, diffs, and verification evidence in one timeline.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Worktrees</div>
  <div class="card-body">Isolate risky or parallel edits without blocking the current checkout.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Side chats</div>
  <div class="card-body">Ask for explanation, critique, or options without derailing the main implementation thread.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Thread resume</div>
  <div class="card-body">Pick work back up with the prior context, checks, and decisions still attached.</div>
</div>

</div>

<p style="margin-top: calc(24 * var(--pt)); color: var(--codex-fg-muted); font-size: calc(15 * var(--pt));">
  Local mode is best when filesystem state, exact commands, and reviewable diffs are the center of gravity.
</p>
