# Context surfaces

<div class="cards-grid cols-3" style="margin-top: calc(18 * var(--pt));">

<div class="card">
  <div class="card-eyebrow">Prompt context</div>
  <div class="card-body">State the outcome, constraints, non-goals, and verification standard. Ambiguity becomes implementation risk.</div>
</div>

<div class="card">
  <div class="card-eyebrow">File mentions</div>
  <div class="card-body">Point at the files that define the boundary. A precise mention can save thousands of irrelevant tokens.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Screenshots and images</div>
  <div class="card-body">Use visual evidence for UI state, design intent, layout regressions, and bugs that text cannot describe cleanly.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Appshots</div>
  <div class="card-body">Capture application state as context so a future step can reason about what was actually visible.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Tool evidence</div>
  <div class="card-body">Command output, browser logs, GitHub checks, and connector reads are evidence, not instructions.</div>
</div>

<div class="card featured">
  <div class="card-eyebrow">Rule</div>
  <div class="card-body">Context should reduce guesswork. If it only adds noise, leave it out.</div>
</div>

</div>
