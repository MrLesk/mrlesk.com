# How to use the tracks

Choose the track based on the part of the loop you want to strengthen.

- Surface selection and setup
- Context design and repo rules
- Tooling and connectors
- Delivery and review discipline

::right::

# The shared standard

By the end of a build track, aim to leave with:

- A working thread or task pattern
- A clear permission model
- A verification recipe
- A diff or artifact someone else can review
