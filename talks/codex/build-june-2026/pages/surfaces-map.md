# Ways to use Codex

<div class="cards-grid cols-3" style="margin-top: calc(18 * var(--pt));">

<div class="card">
  <div class="card-eyebrow">Desktop app</div>
  <div class="card-body">Shared working space for threads, files, screenshots, appshots, browser checks, permissions, and local tools.</div>
</div>

<div class="card">
  <div class="card-eyebrow">CLI</div>
  <div class="card-body">Fastest path for repo-native work: inspect, edit, test, lint, diff, commit, and push from the terminal loop.</div>
</div>

<div class="card">
  <div class="card-eyebrow">IDE extension</div>
  <div class="card-body">Use Codex where code review and editing already happen, with the file tree and editor state close at hand.</div>
</div>

<div class="card">
  <div class="card-eyebrow">Cloud and web</div>
  <div class="card-body">Queue tasks against a repo, review changes, and move heavier work away from a local machine.</div>
</div>

<div class="card">
  <div class="card-eyebrow">ChatGPT mobile</div>
  <div class="card-body">Remote control for starting, checking, and redirecting work when you are away from the development setup.</div>
</div>

<div class="card featured">
  <div class="card-eyebrow">Product idea</div>
  <div class="card-body">The way you start is not the strategy. The strategy is keeping context, permissions, and review close to the task.</div>
</div>

</div>
